# 月次報告作成支援アプリ — プロジェクト概要(引き継ぎ用)

このドキュメントは、PCでのClaude Codeセッションの続きをスマホ(Claudeアプリ)から行うための引き継ぎ資料。詳細な設計判断の経緯はPC側のプランファイルにあるが、要点はここに全て記載している。

## 目的

渡辺晃佑さんが毎月作成しているExcel月報(`報告資料/月報2026年度_渡辺晃佑.xlsx`)の**作成負担を減らす**デスクトップアプリを新規開発する。提出プロセス自体は対象外。

負担が大きい箇所:
1. 報告期間(日付)欄への手入力
2. 1ヶ月前の作業内容を思い出して記入すること
3. 「現場での問題点」「個人の問題点」「次月の目標」の記入

## 制約・前提

- **APIコストをかけない**: アプリ内にAI API呼び出しは一切実装しない。アプリが情報を整理し、ユーザーが手動でClaude.ai(無料版)にコピペして下書きを生成し、結果を貼り戻す運用にする。
- **Teams連携・Slack連携は今回のスコープ外**(将来対応)。理由:
  - Teams: Microsoft Graph APIの認証設定(Azure ADアプリ登録)に社内の許可が必要なため後回し。
  - Slack: 別プロジェクト「進捗管理ToDoアプリ」(`C:\Users\nabe\Desktop\claude\ToDo`)が既にSlack連携済みだが、ElectronのsafeStorage暗号化はアプリごとにOSキーチェーンに紐づく実装のため、ToDoアプリが保存したトークンを本アプリから直接復号できない。トークン再入力の手間も踏まえ見送り。
- **新規・別リポジトリ**: 本アプリはこの「月次報告」リポジトリ(データ管理用)とは別に、新しいフォルダ・新しいGitHubリポジトリとして作成する(例: `monthly-report-app`)。ToDoアプリともmonorepo化しない。**まだリポジトリ自体は作成していない。**
- 技術スタック: Electron + React + TypeScript + `better-sqlite3`(既存のToDoアプリと同じ構成を踏襲)。

## Excelファイルの実データ構造(調査済み・確定)

パス: `報告資料/月報2026年度_渡辺晃佑.xlsx`

### シート構成
月ごとに1シート。表記ゆれに注意: `11月度（2025年度）`(全角括弧)、`12月度(2025)`(半角括弧)、記入済み`1月度`〜`5月度`、未記入`6月度`〜`12月度`、旧レイアウトの`9月度（サンプル）`、解説のみの`書き方`、空の`Sheet9`。「様式 」(末尾半角スペース)が複製元テンプレート。→ シート名は「年度+月」の構造化データとして正規化するパーサーが必要(年度またぎ・表記ゆれ吸収)。

### 上部の基本情報エリア(B1〜C12付近、行番号は内容量で変動しない固定部分)
- Row1: B1見出し「XX月度報告書」/ C1「報告期間：YYYY/MM/DD～YYYY/MM/DD」(**文字列**、Excel日付型ではない)
- Row2: 1セルに「ラベル：値」を直書き。B2「社員番号：...」/ C2「課：...」/ D2「氏名：...」/ E2「報告日：...」
- Row3(見出し)/Row4(値): エンドユーザ・作業場所・経由・出退勤時刻
- Row5(見出し)/Row6(値): システム名・フェーズ・開始年月〜完了予定年月
- Row7(見出し)/Row8(値): 契約形態・契約条件・時間幅の場合の基準時間・契約開始年月〜契約完了年月
- Row9(見出し)/Row10(値): チーム員数・言語/DB/ミドルウェア/プラットフォーム等
- Row11(見出し)/Row12(値、B12:E12マージセル): 作業概要

### 可変長項目(行番号は内容量で変わるため、**B列ラベル文字列の完全一致で位置特定する**設計が必須)

| ヒアリングでの呼称 | 実際のExcelラベル | セル位置 |
|---|---|---|
| 作業内容 | 「作業内容：」 | B18(値はC18〜可変) |
| 現場の問題 | 「現場での問題点：」 | B29(値はC29〜可変) |
| 個人の問題 | 「個人の問題点：」 | B34(値はC34〜可変) |
| 次月の課題 | 「月間目標：」配下の「[次月の目標]」 | C48(固定サブラベル型) |

「現場の展望：」(B39)・「[今月の目標]」(C44)・「[達成度]」(C46)は自動下書き対象外、手動入力項目。

**境界判定は既知ラベル文字列の集合との完全一致のみで行う**(「次にB列に何か出たら」ではない。実データはB列に自由記述が入る月もあるため)。

## 採用する設計方針(確定済み)

1. **Excel読み書き**: `exceljs`採用(マージセル・スタイル保持のため)。要検証: ブック全体往復で図形・customXml・印刷設定・条件付き書式が壊れないか(実データの`9月度（サンプル）`に図形あり)。壊れる場合はXMLレベルの部分置換にフォールバック。
2. **シート複製**: 対象月シートがなければ「様式 」シートを複製(exceljsに直接APIがないため自前実装)。
3. **日付自動計算**: 前月シートのC1報告期間をパースし終了日の翌日を開始日候補に自動算出。前月シートが無ければ手動入力にフォールバック。
4. **ToDoアプリ連携**(`C:\Users\nabe\Desktop\claude\ToDo`、better-sqlite3・DBは`%APPDATA%\todo-app\todo.db`):
   - 読み取り専用接続。WAL同時アクセスのロック挙動は要検証。
   - **前提修正が必要**: `src/main/db.ts`の`COMPLETED_PURGE_DAYS=15`が完了タスクを15日で削除する設定になっており、月末に月報を書く運用だと必要なタスクが消えている可能性が高い。→ **45日以上に延長する設定変更を、本アプリ開発着手前の前提作業として実施する**(Phase 0)。
5. **下書き生成フロー(一括プロンプト方式)**: 「作業内容」「現場での問題点」「個人の問題点」「次月の目標」4項目分をまとめた1つのプロンプトを自動生成(ToDoの完了/繰越タスク＋前月シートの同項目内容が材料)。ユーザーがClaude.ai(無料版)に貼り付けて生成→結果を1回貼り戻す→見出しをもとにアプリが4項目へ自動振り分け。APIは一切呼ばない。
6. **前月参照**: 現場での問題点・個人の問題点・次月の目標について、前月シートの同じ欄を折りたたみ表示で参考提示。
7. **エラー・安全設計(MVP必須)**: 保存前の自動バックアップ、一時ファイル書き込み→アトミックリネーム、Excelファイルが他プロセスで開かれている場合のロック検知エラー表示。

## UIモックアップの現状

Artifact URL: `https://claude.ai/code/artifact/455d9021-0cb1-4f30-a1e8-3ca789f45456`(ファイル: `ui-mockup.html`)

- Electronデスクトップアプリのウィンドウを模した見た目(タイトルバー・Windows風キャプションボタン)。
- タイトルバーに配色切り替え(紺白/緑白ライトモード)のスウォッチボタンと、氏名入力欄(保存ファイル名に使用)を配置。
- 常時表示の期間バー(画面A): 対象月選択、報告期間(自動算出+手動上書き可)、ToDo連携ステータス。
- タブ構成:
  - 「作業報告(下書き対象)」タブ: 一括プロンプト生成パネル + 4項目(作業内容/現場での問題点/個人の問題点/次月の目標)の下書き編集カード。
  - 「現場情報」タブ: 基本情報(社員番号・課・エンドユーザ・作業場所・経由・出退勤時刻・システム名・フェーズ・契約関連・チーム員数・言語DB等、実データ準拠)+ 作業概要 + 現場の展望・今月の目標・達成度(手動入力のみ)。
- 常時表示の保存バー: 保存先ファイル名表示、未保存警告、「Excelに保存(自動バックアップ作成)」ボタン。
